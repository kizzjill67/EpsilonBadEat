# 📘 Инструкция по установке и настройке плагина EpsilonBadEat

## 📥 Установка

1. **Скопируйте файл плагина** `EpsilonBadEat-1.0.0.jar` в папку `plugins` вашего сервера.
2. **Перезапустите сервер** или выполните команду `/reload confirm` (не рекомендуется на продакшене).
3. После первого запуска в папке `plugins/EpsilonBadEat/` будет создан файл `config.yml`.

## ⚙️ Настройка

Откройте файл `config.yml` и настройте параметры под ваш сервер:

### Основные настройки

```yaml
# Как часто проверять еду на порчу (в тиках, 20 тиков = 1 секунда)
check-interval-ticks: 400

# Проверять только игроков онлайн (если false, еда портится даже оффлайн)
check-online-players-only: true
```

### Сроки годности еды

В секции `food-shelf-life` укажите срок годности для каждого типа еды:

```yaml
food-shelf-life:
  BREAD: 2d        # 2 дня
  COOKED_BEEF: 12h # 12 часов
  APPLE: 3d        # 3 дня
  CARROT: 5d       # 5 дней
  # Форматы: Xm (минуты), Xh (часы), Xd (дни), 0 (бесконечно)
```

### Настройки «бочки» (холодильника)

«Бочка» — это контейнер (сундук, бочка), над которым расположено определённое количество блоков (например, земля или камень). В таких контейнерах еда портится медленнее.

```yaml
storage:
  enabled: true           # Включить функцию
  height: 3               # Сколько блоков должно быть над контейнером
  required-blocks:        # Какие блоки подходят для «холодильника»
    - STONE
    - COBBLESTONE
    - DIRT
  multiplier: 2.0         # Во сколько раз дольше хранится еда
  container-types:        # Типы контейнеров
    - CHEST
    - BARREL
    - SHULKER_BOX
```

**Пример постройки холодильника:**
```
[Земля]
[Земля]
[Земля]
[Сундук]
```

### Испорченная еда

Настройте предмет, в который превращается испорченная еда:

```yaml
spoiled-item:
  material: ROTTEN_FLESH          # Материал предмета
  display-name: "&cИспорченная еда" # Название (поддерживает цвета)
  lore:                            # Описание
    - "&7Непригодна к употреблению"
```

### Интеграция с EpsilonDoctor

Если у вас установлен плагин **EpsilonDoctor**, настройте применение болезней при поедании испорченной еды:

```yaml
epsilon-doctor:
  enabled: true                              # Включить интеграцию
  disease-on-spoiled: "пищевое_отравление"   # Название болезни
  method: "COMMAND"                          # COMMAND или API
  command: "doctor infect %player% пищевое_отравление"
```

### Сообщения

Настройте сообщения, которые видит игрок:

```yaml
messages:
  prefix: "&7[&cЕда&7] "
  spoiled-food-eat: "&cВы попытались съесть испорченную еду! Вы заболели."
  spoiled-food-replace: "&cВаша еда испортилась и превратилась в гнилую плоть."
  food-check: "&aСрок годности: &e%time_left%"
  food-check-fresh: "&aЕда свежая."
  no-permission: "&cУ вас нет прав на эту команду!"
  config-reloaded: "&aКонфигурация перезагружена!"
  not-food: "&cЭто не еда!"
```

## 🎮 Команды

| Команда | Описание | Право доступа |
|---------|----------|---------------|
| `/еда проверить` | Показывает оставшийся срок годности предмета в руке | `epsilonbadeat.use` |
| `/еда перезагрузить` | Перезагружает конфигурацию | `epsilonbadeat.admin` |
| `/edible check` | Английский алиас команды проверки | `epsilonbadeat.use` |
| `/food check` | Английский алиас команды проверки | `epsilonbadeat.use` |

## 🔧 Права доступа

```yaml
epsilonbadeat.use:    # Доступ к команде проверки (по умолчанию у всех)
  default: true

epsilonbadeat.admin:  # Доступ к административным командам (по умолчанию у OP)
  default: op
```

## 🍞 Как это работает

1. **Получение еды**: Когда игрок получает еду (крафт, лут, покупка), плагин добавляет к предмету метку времени создания.
2. **Проверка порчи**: Каждые N тиков плагин проверяет, не истёк ли срок годности еды.
3. **Превращение**: Если срок истёк, еда заменяется на гнилую плоть с названием «Испорченная еда».
4. **Поедание**: При попытке съесть испорченную еду игрок получает болезнь (через EpsilonDoctor) и предмет заменяется на гнилую плоть.
5. **Хранение в бочке**: Если еда лежит в контейнере под слоем блоков (земля, камень и т.д.), срок её годности увеличивается в указанное количество раз.

## 🛠 Примеры использования

### Создание холодильника

1. Выкопайте яму глубиной 1 блок.
2. Поставьте в неё сундук или бочку.
3. Засыпьте сверху 3 блоками земли или камня.
4. Готово! Еда в таком контейнере будет портиться в 2 раза медленнее.

### Проверка срока годности

1. Возьмите предмет еды в руку.
2. Выполните команду `/еда проверить`.
3. Вы увидите оставшееся время до порчи.

## ❓ Частые вопросы

**Q: Еда не портится?**
A: Убедитесь, что:
- Предмет есть в списке `food-shelf-life` в конфиге.
- Прошло достаточно времени.
- Сервер не был перезапущен (метаданные предметов сохраняются).

**Q: Можно добавить кастомную еду из ItemsAdder?**
A: Да, добавьте ID предмета в формате `itemsadder:your_item_id: 3d` в секцию `food-shelf-life`.

**Q: Как отключить порчу определённого предмета?**
A: Установите для него значение `0` в конфиге (например, `ENCHANTED_GOLDEN_APPLE: 0`).

**Q: Еда портится слишком быстро/медленно?**
A: Измените значения в секции `food-shelf-life` и выполните `/еда перезагрузить`.

## 📞 Поддержка

При возникновении проблем:
1. Проверьте консоль сервера на наличие ошибок.
2. Убедитесь, что версия сервера совместима (Paper/Purpur 1.20.4+).
3. Убедитесь, что плагин EpsilonDoctor установлен (если используете интеграцию).
